# 项目记忆模板

## 使用说明

在创作每部小说时，在项目的 `memory/` 目录下创建以下文件，
Claude将会在每次对话时读取这些文件，保持创作的连贯性。

---

## 使用流程

### 新建项目时

1. 在项目目录创建 `memory/` 文件夹
2. 根据上述模板创建对应文件
3. 填写基础信息（角色、大纲、设定）
4. 开始创作，Claude会自动读取这些文件

### 创作过程中

1. **每次对话前**：Claude会读取memory文件，了解项目当前状态
2. **每次创作后**：根据新增的情节更新相关文件
   - 新角色 → 更新 characters.md
   - 新情节点 → 更新 plot_threads.md
   - 进度变化 → 更新 writing_progress.md
3. **定期回顾**：每完成一个章节或情节段落后，检查记忆文件是否需要更新

### 示例工作流

```
用户："帮我续写第3章，林晓和张明的会议场景"

Claude：
1. 读取 characters.md 了解两人关系和性格
2. 读取 plot_threads.md 了解当前情节进展
3. 读取 style_notes.md 了解写作风格偏好
4. 创作场景
5. 提醒用户：是否需要更新某个记忆文件

用户："写得不错，继续"
[Claude继续，保持一致性]
```

## 技巧提示

### 记忆文件的颗粒度

**太少**：只有角色名字 → Claude无法保持一致性
**太多**：每句对话都记录 → 文件臃肿，难以维护
**适中**：记录关键信息、转折点、核心设定

### 更新频率

- **高频更新**：plot_threads.md（每次创作后）
- **中频更新**：writing_progress.md（每章完成后）
- **低频更新**：characters.md, worldview.md（有新增或重大变化时）
- **偶尔更新**：style_notes.md（发现新的风格偏好时）

### 跨会话连续性

因为有memory文件，即使隔了几天甚至几周，
只要文件保持更新，Claude就能无缝接续之前的创作。

### 协作创作

如果有多个作者，memory文件可以作为共享的"世界观圣经"，
确保所有人的理解一致。

## 常见问题

**Q: 我一定要创建所有文件吗？**
A: 不一定。根据项目复杂度选择：
- 简单短篇：只需 characters.md 和 writing_progress.md
- 中篇小说：加上 outline.md 和 plot_threads.md
- 长篇小说：建议全部创建

**Q: 如果我的故事发生了大改动怎么办？**
A: 更新memory文件！记忆系统的优势就是可以随时调整。

**Q: Claude会自动更新这些文件吗？**
A: Claude会在创作后提醒你需要更新哪些文件，
但实际编辑需要你确认（保证创作主动权在作者手中）。

**Q: 文件太长怎么办？**
A: 可以分拆，例如 characters.md 分为 characters_main.md 和 characters_supporting.md
